System Architecture & Overview

Vintage Radio V2

The Vintage Radio V2 is a Service-Oriented Architecture (SOA) engineered for the ESP32-WROVER-E. It effectively bridges 1930s mechanical aesthetics with modern high-fidelity audio capabilities by decoupling hardware sensing, audio decoding, and system orchestration.

"The software architecture prioritizes stability by isolating time-critical tasks into dedicated FreeRTOS tasks to ensure zero-stutter playback."

1. Audio & Streaming Orchestration

The audio engine provides a unified interface for diverse streams, utilizing a shared I2S bus.

  • Hardware Arbitration: The AudioService acts as an owner-manager, ensuring only one mode (Bluetooth, MP3, or Network) controls the I2S hardware at any given time.
  • Safe Playback Path: A custom AudioInfoGuard prevents driver crashes by filtering zero-sample-rate metadata generated during decoder initialization.
  • Tuning Simulation: A "Wait Tone" (simulated radio static) bridges digital latency during station switches or Wi-Fi buffering to maintain the vintage user experience.

2. Sensing & Interaction Logic

Interaction is digitized while maintaining analog smoothness through sophisticated signal processing.

  • Exponential Moving Average (EMA): High-frequency ADC noise from potentiometers is filtered using EMA to ensure volume and tuning stability.
  • Multilayer Input Queue: The InputService decodes quadrature signals and identifies Short, Long, and Very Long button presses to allow for secondary calibration functions.
