Bluetooth Mode Technical Deep Dive

This module implements an A2DP Sink (receiver) for the ESP32, allowing the device to act as a high-quality Bluetooth speaker.

1. I2S Configuration for Audio

Bluetooth audio requires a specific I2S setup to handle the incoming data stream from the smartphone.

  • DMA Management: Uses 16 DMA buffers of 512 bytes each. This specific configuration was chosen as a stable profile to prevent boot-loop regressions.
  • Clocks: Configured for 44.1 kHz, 16-bit stereo. tx_desc_auto_clear is enabled to ensure the I2S hardware outputs silence if the Bluetooth buffer runs dry, preventing harsh digital noise.

2. Lifecycle Management

modeBtStart()

Initializes the Bluetooth stack and audio hardware.

  • WiFi Conflict Mitigation: Explicitly turns off WiFi (WIFI_OFF) to prevent 2.4GHz interference and ensure the Bluetooth radio has full priority.
  • I2S Acquisition: Requests hardware ownership via audioStartBtMode().
  • Persistence: Uses the Preferences (NVS) library to look up the last connected device's MAC address. If found, it attempts an automatic reconnection; otherwise, it enters discoverable mode.

modeBtStop()

Performs a "safe stop" to allow for clean reboots or mode switching.

  • Stopping Logic: Sets a s_stopping flag and disables auto-reconnect to prevent the Bluetooth stack from trying to reconnect while the system is shutting down.
  • Resource Release: Closes AVRCP and A2DP connections before calling audioReleaseBt() to return I2S control to the system.

3. Peer Memory (NVS)

save_peer(const esp_bd_addr_t* peer)

When a connection is successfully established, the peer's MAC address is formatted as a string (e.g., 00:1a:2b:3c:4d:5e) and stored in Non-Volatile Storage (NVS). This allows the "BoomBox" to recognize and auto-connect to your phone the next time it powers on.

4. User Interaction (AVRCP)

The module supports standard Bluetooth remote control features via AVRCP:

  • btPlayPauseToggle(): Queries the current audio state. If music is playing, it sends a pause command; otherwise, it sends play.
  • btNext() / btPrev(): Sends track skip commands to the connected source device.
  • btSetVolumePercent(): Maps the 0-100% volume scale to the 0-127 range required by the Bluetooth Absolute Volume standard.

5. Connection States

on_connection_state_changed()

A callback function triggered by the A2DP library.

  • Connected: Triggers save_peer and turns off discoverability to prevent other users from highjacking the stream.
  • Disconnected: If the shutdown process isn't active, it triggers a reconnection attempt and re-opens discoverability.
