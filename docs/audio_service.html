Audio Service Technical Deep Dive

This document details the I2S audio pipeline, MP3 decoding, and hardware management for the TAS5828M amplifier.

1. Architecture & The "Audio Guard"

The service uses the AudioTools library to manage I2S streams. A critical component is the AudioInfoGuard class.

class AudioInfoGuard

  • The Problem: The Helix MP3 decoder often broadcasts empty audio information {0,0,0} during initialization, which can cause the I2S driver to crash or assert.
  • The Solution: This class intercepts setAudioInfo calls and only passes valid configurations (sample rate, channels, bits) to the physical I2S hardware.
  • Hardware Sync: Whenever the audio format changes, it triggers audioEnsurePlaybackPath() to clear any amplifier clock faults caused by the momentary I2S clock halt.

2. Ownership & Resource Management

Because the I2S hardware is a shared resource, the service uses an I2SOwner arbitration system.

audioAcquireLocal() / audioAcquireBt() / audioAcquireNet()

  • Arbitration: Only one mode can control the I2S bus at a time. If Bluetooth or Network mode is active, Local (SD) playback is denied unless explicitly released.
  • Dynamic Start: I2S is only initialized (s_i2s.begin) when a mode successfully acquires the hardware.
  • Task Management: audioAcquireLocal spawns a dedicated FreeRTOS task (audioTask) on Core 1 to handle MP3 decoding without stuttering.

3. Local Playback Logic

audioPlay(const char* path)

This function manages the transition from idle to active music playback.

  • Wait Tone Override: If the system plays /system/sound/wait.mp3, it applies a specific volume percentage (s_waitVolume) loaded from the SD card config.
  • Decoder Setup: It opens the file from the SD card and initializes the EncodedAudioStream with the Helix MP3 decoder.
  • State Control: Sets s_busy = true, which signals the background task to start moving data.

audioStop()

Safely shuts down the stream. It includes a safety loop that waits up to 150ms for the background task to exit the copy() block before closing files and freeing decoder memory.

4. Amplifier Control

audioSetVolume(uint8_t percent)

Calculates the "actual" volume by considering the master volume and whether a special tone (like the wait tone) is currently playing. It communicates this to the TAS5828M driver via I2C.

audioEnsurePlaybackPath()

A vital recovery function that wakes the amplifier from its power-down state and clears any "Clock Fault" registers that occur when I2S clocks start or change frequency.

5. The Audio Task

audioTask(void*)

A high-priority loop running every 2ms.

  • Data Pumping: Calls s_copier.copy() to move decoded MP3 data from the decoder output to the I2S buffer.
  • Auto-Stop: If copy() returns 0 or less, the task detects the End-Of-File (EOF) and automatically calls audioStop().
  • Clean Teardown: Monitors s_teardownLocalTask to allow the task to delete itself safely when switching modes.
