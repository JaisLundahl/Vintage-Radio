Hardware Pinmap (ESP32-WROOM-32E)

The pins.h file defines the static hardware configuration. It ensures all services reference the same physical GPIO pins for communication and sensing.

1. Audio and Communication Interfaces

The system uses three primary digital buses to communicate with the amplifier and storage.

Interface Signals GPIO Pins
I2S (Audio Data) BCK, LRCK, DATA 14, 25, 27
I2C (Amp Config) SDA, SCL 21, 22
SPI (SD Card) CS, SCK, MOSI, MISO 5, 18, 23, 19

2. Analog Inputs (Sensors)

Analog inputs are mapped to ADC1 pins because ADC2 cannot be used simultaneously with WiFi.

  • PIN_VOL_POT (GPIO 36): Dedicated to the volume control potentiometer.
  • PIN_TUNER_POT (GPIO 39): Dedicated to the station tuning potentiometer.
  • PIN_VBAT_ADC (GPIO 34): Used for battery voltage monitoring via a voltage divider.
  • PIN_MODE_SEL_ADC (GPIO 35): Used for the 3-way mode switch (BT/MP3/NET).

3. User Interface & Feedback

Standard GPIOs are utilized for the rotary encoder and the visual status LED.

  • Rotary Encoder: Signals A (GPIO 32), B (GPIO 33), and Push Button (GPIO 13).
  • RGB LED: PWM-capable pins for Red (GPIO 26), Green (GPIO 2), and Blue (GPIO 4).

4. Software Configuration Flags

This file also acts as a compile-time configuration hub.

CURRENT_INPUT_MODE: Currently set to INPUT_MODE_ROTARY, which instructs the InputService to treat PIN_ENC_A and PIN_ENC_B as quadrature signals rather than independent buttons.
