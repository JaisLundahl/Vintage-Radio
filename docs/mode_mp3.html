MP3 Mode Technical Deep Dive

This module implements a virtual "Radio" experience using local MP3 files, featuring folder-based stations, history tracking, and a simulated tuning effect.

1. Virtual Radio Tuning Logic

The MP3 mode simulates an analog radio using the Tuner potentiometer via the AdcService.

  • Tuning Effect: When the tuner moves to a new "bin" (station), the system immediately plays a "static" or wait tone (wait.mp3).
  • Confirmation Delay: The system waits 1.5 seconds (kTuneConfirmMs) before committing to a folder. This prevents rapid-fire file access while the user is still turning the knob.
  • Station Mapping: 16 folders (e.g., /folder1 to /folder16) are mapped to 16 ADC bins.

2. Playback History & Navigation

Unlike a simple random player, this module maintains a sophisticated history per folder.

FolderState Structure: Stores the current path, lastIdx (current track), maxIdx (total tracks found), and a history vector of the last 16 played songs.
  • modeMp3Next(): If the user is currently browsing backward through history, "Next" moves forward in that history. If they are at the end, it generates a new random track.
  • modeMp3Prev(): Allows the user to step back through the specific sequence of random songs played in the current folder.
  • Fallback Mechanism: If a calculated file (e.g., 00000005.mp3) is missing, the system searches backward up to 20 files (kMaxSearchDepth) to find the nearest playable track.

3. Persistence and State Management

To ensure the "Radio" remembers where it left off, state is saved to the SD card.

writeGlobalStateFile()

  • Safe Writes: Uses a s_writingState flag to ensure the SD card isn't unmounted or the task killed during a file write operation.
  • Data Stored: Saves the path, last played index, folder capacity, and the full history sequence for all 16 folders.

initOrLoadGlobalState()

  • Auto-Indexing: If no state file is found, the system performs a one-time deep scan of the SD card to discover the maxIdx for every folder.

4. Background Task (mp3Task)

A dedicated FreeRTOS task handles the high-level logic every 50ms.

  • Seamless Wait Tone: Monitors the wait tone and loops it automatically if the user remains between "stations".
  • Auto-Advance: When a song finishes naturally, the task automatically triggers modeMp3Next() to keep the music playing.
  • Tuner Deadzone: Temporarily ignores tuner input (s_ignoreTunerUntil) after a manual track skip to prevent electrical noise from accidentally changing the station.

5. Integration with AudioService

While mode_mp3 decides what to play, it uses AudioService to perform the actual I2S streaming.

  • I2S Ownership: Calls audioStartMp3Mode() to hijack the audio hardware from Bluetooth or Network modes.
  • Control: Uses audioPause() and audioResume() for the Play/Pause toggle to keep the decoder state in memory.
