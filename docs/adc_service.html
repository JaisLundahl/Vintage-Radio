ADC Service Technical Deep Dive

This document details the implementation of Analog-to-Digital Conversion (ADC) processing for Volume, Battery, Tuner, and Mode selection on the ESP32 platform.

1. System Initialization

adcInit()

This function prepares the ADC hardware and spawns the background processing task.

  • Pin Configuration: Sets INPUT mode for the Volume Pot, Tuner Pot, Battery ADC, and Mode Selector.
  • Calibration Persistence: Attempts to open /system/sound/max_volumes.txt from the SD card to load user-calibrated TUNER_MIN_RAW and TUNER_MAX_RAW values.
  • Filter Priming: Performs an initial read to seed the Exponential Moving Average (EMA) filters to prevent value jumping on startup.
  • Task Creation: Uses xTaskCreatePinnedToCore to launch the adcTask on Core 1, running at 100Hz (every 10ms) to ensure smooth filtering independent of the main loop.

2. Data Processing & Filtering

readAdcAveraged(uint8_t pin)

To combat electrical noise, this function performs Multisampling. It reads the analog pin 8 times (SAMPLES_PER_TICK) and returns the average. The bitmask & 0x0FFF is applied to ensure the result stays within the 12-bit range (0-4095).

adcTick()

The heart of the service, executed every 10ms. It processes four distinct signals:

  • Volume: Applies an EMA filter. It also implements Deadbands (1% at the bottom and top) to ensure that the volume stays at a "true zero" or "true 100%" despite potentiometer flickering.
  • Battery: Converts the 12-bit raw value to millivolts ($mV$) using a reference voltage of 3300mV and accounts for the physical Voltage Divider resistor values (33k / 6.8k).
  • Tuner: Maps the raw pot value into 16 discrete "bins." It handles "reversed" calibration (where Min > Max) and applies Hysteresis. A new bin is only accepted if it remains stable for 200ms (TUNER_STABLE_MS).
  • Mode Selector: Simple EMA-filtered raw reading for mode switching logic.

3. Tuner Calibration & Persistence

adcSetTunerMin() / adcSetTunerMax()

These functions allow for "On-the-fly" calibration. When called, they capture the current ADC value of the Tuner potentiometer and save it as the new boundary.

updateTunerConfig()

A helper function that handles SD card file I/O. It reads the existing config file, replaces the specific calibration key (e.g., TUNER_MIN_RAW), and writes the updated string back to the SD card. This ensures calibration survives a power cycle.

4. Mathematical Constants

The service uses specific constants to tune performance:

EMA_SHIFT = 3 Equivalent to an alpha of 1/8. Provides a balance between smooth signal and low latency.
ADC_VREF_MV = 3300 The assumed reference voltage of the ESP32 ADC.
TUNER_STABLE_MS = 200 Debounce time for the tuner to prevent "jittering" between two radio stations or settings.
