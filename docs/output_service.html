Output Service Technical Deep Dive

This service manages the visual feedback system of the device, specifically controlling an RGB LED via hardware PWM (LEDC) and smooth fade transitions.

1. LEDC Hardware PWM Setup

The service utilizes the ESP32's LEDC (LED Control) peripheral to provide flicker-free color mixing.

  • Channels: Three independent channels are assigned for Red (0), Green (1), and Blue (2).
  • Configuration: Initialized with a 5000Hz frequency and 8-bit resolution (0-255 depth).
  • Fade Driver: Calls ledc_fade_func_install(0) to enable the ESP-IDF hardware-assisted fading, allowing for smooth pulses without manual CPU-driven stepping.

3. Pulse Engine Implementation

The pulsing logic provides a "breathing" effect by oscillating duty cycles over time.

startPulseOnChannel(uint8_t ch)

This prepares the pulse by resetting other active pulses (to avoid color clashing) and triggering the initial hardware fade from 0 to 255 over 2 seconds (PULSE_HALF_MS).

servicePulseChannel(uint8_t ch)

Called repeatedly by loop(), this checks the nextSwitchMs timestamp. When reached, it reverses the fade direction (up to down or down to up) and schedules the next hardware fade transition.

4. Low-Level Control

setRGB(uint8_t r, uint8_t g, uint8_t b)

The fundamental function that writes raw 8-bit values directly to the LEDC channels, allowing for any of the 16.7 million colors available in the RGB spectrum.

5. Design Pattern

The service is implemented as a Singleton. This ensures that only one piece of code can control the system LEDs at any time, preventing hardware conflicts. It is accessed globally via the outputService() function.
